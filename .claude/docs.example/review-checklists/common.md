# 공통 코드 리뷰 체크리스트 (스택 무관)

> 모든 dev-* 에이전트가 작업 완료 전 자가검증. code-reviewer가 최종 리뷰.

## 공통 체크

- [ ] 요구사항(Jira 티켓)을 충족하는가
- [ ] null/NPE 처리가 적절한가
- [ ] 예외 처리가 누락된 곳은 없는가
- [ ] 하드코딩된 값이 있는가 (URL, 키, 매직 넘버) — 있다면 상수/설정으로 분리
- [ ] 불필요한 주석이나 디버그 코드(Log.d, System.out.println, console.log)가 남아있는가
- [ ] 변수/함수명이 명확한가 (한 글자 변수, 의미 불명 약어 금지)
- [ ] 본인 변경이 만든 미사용 import/변수/함수가 남아있지 않은가
- [ ] **CLAUDE.md "Surgical Changes" 원칙 준수** — 인접 코드 "개선"하지 않았는가
- [ ] 시크릿/민감정보가 로그에 출력되지 않는가 (`@.claude/docs/security/secret-handling.md` 참조)

## 심각도 분류

리뷰 코멘트는 다음 레벨로 분류:

- **[CRITICAL]** — 보안 취약점, 데이터 손실 가능, 운영 장애 유발 가능. 반드시 수정.
- **[MAJOR]** — 기능 오동작, 메모리 누수, 성능 문제. 수정 권장.
- **[MINOR]** — 코드 스타일, 가독성, 경미한 개선. 선택적 수정.
- **[INFO]** — 제안, 리팩토링 아이디어. 참고용.

## 출력 형식

```
## 코드 리뷰 결과

### 총평
[전반적인 평가 한 줄]

### 발견 사항

**[CRITICAL] 파일명:라인번호**
- 문제: [설명]
- 수정안: [코드 또는 설명]

**[MAJOR] 파일명:라인번호**
- 문제: [설명]
- 수정안: [코드 또는 설명]

### 잘된 점
- [긍정적인 부분]

### 수정 필요 항목 요약
- [ ] [CRITICAL] ...
- [ ] [MAJOR] ...
```

## 자가검증 / 최종 리뷰 절차

1. 변경 파일 목록 파악 (`git diff --name-only`)
2. 스택에 맞는 체크리스트 추가 적용
3. Jira 티켓 키가 있으면 `jira_ticket_detail`로 요구사항 재확인
4. 위 출력 형식대로 결과 작성
5. CRITICAL/MAJOR 항목 있으면 즉시 수정
6. 요청 시 `jira_add_comment`로 리뷰 결과 티켓 댓글 등록
